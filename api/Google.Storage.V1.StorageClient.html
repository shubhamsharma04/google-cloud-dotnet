Class StorageClient

Abstract class providing operations for Google Cloud Storage.

Note: beta release
This is a beta release of the Cloud Storage API.

Inheritance
System.Object
StorageClient
Namespace:Google.Storage.V1
Assembly:Google.Storage.V1.dll
Syntax
public abstract class StorageClient
Remarks
This abstract class is provided to enable testability while permitting additional operations to be added in the future. See Create(GoogleCredential) and CreateAsync(GoogleCredential) to construct instances; alternatively, you can construct a StorageClientImpl directly from a Google.Apis.Storage.v1.StorageService. All instance methods declared in this class are virtual, with an implementation which simply throws System.NotImplementedException. All these methods are overridden in StorageClientImpl.

Service

The underlying storage service object used by this client.

Remarks

The StorageClient class only provides convenience operations built on top of an existing service object. Any more complex operations which are not supported by this wrapper may wish to use the same service object as the wrapper, in order to take advantage of its configuration (for authentication, application naming etc).

DeleteObject(Object, DeleteObjectOptions)

Deletes a version of the specified object synchronously.

Remarks
By default this will delete the latest version of the object, but this can be controlled using Generation. Note that any generation information contained within obj is ignored; the generation to delete is only controlled via options. For buckets that support multiple versions, implicitly deleting the latest version only archives it so the object is still available and can be listed by specifying Versions. If the version is explicitly specified, the object is permanently deleted. If the bucket containing the object supports versioning, after this operation completes successfully there may still be another version of the same object. In buckets which do not support versioning, this operation will permanently delete the object. If the request attempts to delete an object that does not exist or a specific version that does not exist, this counts as a failure with an HTTP status code of 404.
